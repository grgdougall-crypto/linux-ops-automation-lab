# Linux Ops Automation Lab

A Linux systems administration and automation project built on Debian Linux. This project demonstrates Bash scripting, Python automation, scheduled task execution, system monitoring, reporting, data analysis, and dashboard development using Flask.

## Overview

The Linux Ops Automation Lab simulates operational tasks commonly performed by system administrators, IT support professionals, and security analysts.

The project evolved from a collection of Bash monitoring scripts into a lightweight operations monitoring platform that includes automated workflows, Python-based analytics, health scoring, trend analysis, and a web-based dashboard.

## Scripts

### check_disk_usage.sh

Displays current disk utilization.

### check_memory_usage.sh

Displays current memory and swap utilization.

### service_status_check.sh

Checks the status of key Linux services:

* SSH
* NetworkManager
* Cron

### check_failed_logins.sh

Reviews authentication activity and reports failed login attempts.

### backup_home_directory.sh

Creates compressed backup archives.

### generate_system_report.sh

Creates a consolidated system health report.

### cleanup_old_reports.sh

Applies report retention policies and removes older reports.

### run_daily_automation.sh

Runs the complete automation workflow:

1. Generate system report
2. Calculate health score
3. Perform trend analysis
4. Clean up older reports
5. Write execution log

---

## Python Components

### report_parser.py

Parses generated reports and produces system health summaries.

### health_score.py

Calculates a system health score based on report contents.

### trend_analysis.py

Compares historical reports to identify trends in resource utilization and system health.
